# Two-Factor Authentication

---
## Description

An application allows generating one-time use codes.

- The client application, which must be configured for 2FA, generates a unique identifier: barcode or a character string
- An entry is added in the application managing the 2FA, with the URL of the client application and this identifier
- The application managing the 2FA generates a code, to be entered in the client application, to validate the association between the two
- Each time the client application is launched, a validation code is requested. This code must be generated by the 2FA application.

---
## Application for PC: KeePassXC

Application running on Linux, MacOS, and Windows: [https://keepassxc.org/](https://keepassxc.org/)

For more informations on KeePassXC : [Manage your passwords (KeePass)](./keepass.md)

How it works :

- Install KeePassXC
- Create a new entry for the application
- Once the new entry is created, configure the TOTP function for this entry
    - Right-click, `"TOTP / Setup TOTP ...`"
    - To configure TOTP, provide the client application's code (often associated with the QR Code)
- Once TOTP is configured, display a code:
    - Right-click, `TOTP / Show TOTP ...`
    - Then copy it into the client application's configuration, in the "Authentication code" input field

More information on using TOTP with KeePassXC: [https://www.it-connect.fr/mfa-comment-configurer-le-totp-dans-keepassxc/](https://www.it-connect.fr/mfa-comment-configurer-le-totp-dans-keepassxc/)

## Application for Smartphone: Twilio Authy Authenticator

Install the application.

Configure it

- Enter your phone number
- Enter your email
- A validation SMS and email are sent

Once the configuration is validated, create an "Account" for each client application.
